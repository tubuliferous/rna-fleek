▼ Navigation

3D View uses an orbit camera — you rotate around a center point marked by a small white cross (dark in light theme). The cross has a fixed world size, so zooming makes it grow on screen while flying keeps it the same apparent size. This is intentional: it tells you whether you're changing orbit distance or translating through space. Points closer to the camera appear larger (perspective depth).

The orbit center is constrained inside a bounding sphere computed from the point cloud. You can fly through the entire cloud but the center can't leave it, keeping the view grounded. Zoom-out is capped at 4× the cloud radius.

On first load, the camera performs a brief intro orbit — starting close and pulling back with a half-rotation — to convey that the space is 3D and interactive. Any click or scroll cancels it.

2D View uses an orthographic camera with pan and zoom. Panning is limited so the cloud can't be dragged fully off-screen. Zoom-out shows about 2× the cloud extent; zoom-in goes to 500×. A crosshair appears at screen center during panning to aid orientation.

▼ Selection Groups

Cells are assigned to the active group (highlighted row). Groups are non-overlapping: adding a cell to one group removes it from any other.

Highlight modes (Settings): Ring (default) draws a colored ring around selected cells, preserving the underlying cluster/expression color. Fill replaces the cell color entirely with the group color and dims non-selected cells.

Background Dim slider (View panel): controls how much non-selected cells are faded when selections are visible. Auto-sets on first highlight; adjustable afterward. Selected cells are never dimmed.

▼ Gene Expression

Switch to Gene Expression mode in the Color By section or press G. Press C to switch back to cell type coloring. Type a gene name in the search box — results come from the server's full gene list. Click a gene to load its expression on the UMAP.

Expression values are fetched on demand from the server and cached client-side. Values are normalized 0–1 using the 98th percentile of expressing cells, then mapped to a colorscale: Viridis (dark mode) or Blues sequential (light mode, where low expression fades into the white background).

Top Variable Genes — shown as clickable chips below search. These are either pre-annotated HVGs from the dataset (adata.var['highly_variable']) or the top 2,000 genes by variance, computed on a 50k-cell subsample in the background.

Gene index performance — on load, a CSC (compressed sparse column) index is built in the background for fast per-gene lookups. The spinner in the Data panel shows progress. Before the index completes, gene lookups use a slower row-scan fallback but still work.

▼ DEG Analysis

The DEG button (enabled when 2+ groups have cells) opens the analysis panel at the bottom. Select two groups to compare, choose a test, and click Run.

Tests available:

TestDescription
WilcoxonRank-sum test. Non-parametric, robust for scRNA-seq. Recommended default.
t-testWelch's t-test. Faster, assumes roughly normal expression in each group.

Gene filtering: Before testing, genes expressed in fewer than 3 cells across both groups are removed. This typically reduces ~60k genes to ~10–20k, giving a 3–6× speedup without affecting results for meaningful genes.

Normalization: If the data appears to contain raw counts (integer values with max > 5 in a sample), the server auto-applies normalize_total(target_sum=1e4) and log1p to the subset before testing.

Volcano plot: log₂ fold-change on x-axis. Y-axis toggles between −log₁₀(padj) and |Cohen's d|. Significance thresholds: |log₂FC| ≥ 0.5 and padj < 0.05. Hover to see gene stats; click a gene to load its expression on the UMAP.

Cohen's d is computed directly on the sparse matrix using E[X²]−E[X]² with Bessel correction, avoiding densification.

Caching: Results are cached by test type + group combination. Switching between Wilcoxon and t-test shows cached results instantly if available.

Pseudoreplication caveat: Cells from the same donor/sample are not independent observations. P-values from single-cell comparisons are anti-conservative — they reflect intra-sample variability, not inter-donor. For publication-level results, use pseudobulk approaches.

▼ Data Management

RNA-FLEEK loads .h5ad files (AnnData format). Click the upload icon to select a file. Before loading, a popover lets you set:

OptionDescription
QuickMapSubsample-and-project UMAP. Fits UMAP on a subset (default 50k cells), then projects the rest via PCA → nearest-neighbor. Much faster for large datasets with near-identical quality.
Disk-backedKeeps the expression matrix on disk (backed mode). Slower gene lookups but allows datasets larger than available RAM.

After loading, status badges in the Data panel show which options are active. UMAP coordinates are cached to disk (.umap_cache.npz) for instant reload.

Server startup: python fleek_server.py [file.h5ad] — or start without a file and upload via the UI. The server uses a threading HTTP server for concurrent progress polling during heavy computation.

Exports: Click the save icon on a selection group to download that cell subset as an .h5ad file. Large exports are streamed in 4MB chunks.

▼ Methods & Statistics

UMAP is computed via scanpy using default parameters (n_neighbors=15, min_dist=0.5). Both 2D and 3D embeddings are computed and cached. QuickMap fits on a stratified subsample (preserving cluster proportions) and projects remaining cells via batch PCA and UMAP transform.

Differential expression uses scanpy's rank_genes_groups. P-values are adjusted for multiple testing using Benjamini-Hochberg. Genes expressed in fewer than 3 cells are filtered before testing.

Cohen's d (effect size) is computed as the difference in means divided by pooled standard deviation, with Bessel correction. Computed directly on the sparse matrix via E[X²]−E[X]² to avoid materializing dense arrays.

Expression normalization for visualization uses 98th percentile of expressing cells as the ceiling, mapping to [0, 1]. This prevents a handful of extreme outliers from washing out the color range.

Point rendering uses Three.js WebGL with circle-textured point sprites. In 3D, sizeAttenuation provides perspective-correct depth. In 2D, sizes are manually scaled by zoom level. Point opacity auto-scales inversely with density to prevent over-saturation on large datasets.

Performance notes: The CSC (compressed sparse column) gene index takes ~5 minutes on 1.6M cells but makes individual gene lookups instant. All heavy computation runs in background threads — the visualization is interactive immediately. DEG analysis pre-filters to expressed genes for 3–6× speedup.
